BEFORE PRESENTATION - CHECKLIST (Do 1 day before!)
═══════════════════════════════════════════════════

□ Get API Keys:
  □ NewsAPI key from https://newsapi.org/register
  □ Gemini API key from https://aistudio.google.com/apikey
  
□ Setup:
  □ Create folder "news-analyzer"
  □ Add files: news_analyzer.py, requirements.txt, .env
  □ Create .env with your API keys
  
□ Install:
  □ Run: pip install -r requirements.txt
  □ Wait for all downloads (2-3 minutes)
  
□ Test the app:
  □ Run: streamlit run news_analyzer.py
  □ Wait for "Streamlit server is now running"
  □ Browser opens automatically
  □ Try searching for "Technology news"
  □ Wait 30 seconds for results
  □ Check if summaries work
  
□ Test different searches:
  □ Search 1: "Tesla" (should have good sources)
  □ Search 2: "Bitcoin" (mix of good and bad sources)
  □ Search 3: "Breaking news" (current events)
  
□ Verify features:
  □ Credibility scores show (80%, 45%, etc.)
  □ Sentiment shows (😊, 😢, 😐)
  □ Images display properly
  □ Summaries are 1-2 sentences
  □ Links work when clicked
  □ Sorting works (by credibility/date/sentiment)
  □ Statistics dashboard displays correctly
  
□ Take screenshots:
  □ Successful search result (for backup if internet fails)
  □ Example with high credibility article
  □ Example with low credibility article
  □ Statistics dashboard
  
□ Prepare talking points:
  □ Write down main points from PRESENTATION_SCRIPT.txt
  □ Memorize opening (30 seconds)
  □ Memorize demo flow (know what to click)
  □ Prepare 1-2 current event searches for demo
  □ Practice 2x before bed

COMMON ISSUES & FIXES
════════════════════

Issue: "ModuleNotFoundError: No module named 'streamlit'"
──────────────────────────────────────────────────────
Cause: Dependencies not installed
Fix:
  1. Open command prompt in news-analyzer folder
  2. Run: pip install -r requirements.txt
  3. Wait 3-5 minutes for installation
  4. Try again: streamlit run news_analyzer.py

Issue: "NoneType object is not subscriptable" or similar error
────────────────────────────────────────────────────────────
Cause: .env file not found or API keys not set
Fix:
  1. Check that .env file exists in same folder as news_analyzer.py
  2. Check file contents: NEWSAPI_KEY=xxxxx (no spaces after =)
  3. Make sure keys are pasted correctly (no extra characters)
  4. Restart app: streamlit run news_analyzer.py

Issue: "Connection error" or "Network error"
───────────────────────────────────────────
Cause: Internet connection or API down
Fix:
  1. Check your internet: ping google.com
  2. Try searching again (give APIs time to respond)
  3. Try different search term
  4. If persistent: Use screenshots and speak through features

Issue: "API key not found" message in app
───────────────────────────────────────
Cause: NEWSAPI_KEY not in .env
Fix:
  1. Verify .env file exists in the folder
  2. Open .env with text editor (Notepad)
  3. Ensure first line is: NEWSAPI_KEY=your_actual_key
  4. Save file
  5. Restart app

Issue: App loads but shows no articles
─────────────────────────────────────
Cause: API issue or no articles found
Fix:
  1. Wait 10 seconds (APIs might be slow)
  2. Try different search term (try "technology")
  3. Check if summaries are enabled (might be slow)
  4. Turn off summaries: uncheck "Generate AI Summaries"
  5. Try again

Issue: "Summaries look weird or incomplete"
──────────────────────────────────────────
Cause: Gemini AI sometimes makes errors
Fix: Completely normal! AI can hallucinate or be inaccurate
  1. This is expected behavior
  2. Say in presentation: "AI isn't perfect, but it's fast"
  3. Show that credibility score catches bad sources
  4. Emphasize human review still needed

Issue: App is very slow
──────────────────────
Cause: Generating summaries for 10 articles (20 seconds!)
Fix:
  1. Turn OFF "Generate AI Summaries" checkbox
  2. Or run same search twice (uses cache, instant)
  3. Or search something trending (News API is faster)

Issue: Presenter mode - need to practice
────────────────────────────────────────
Tip:
  1. Memorize exactly what to click
  2. Hotkeys: Ctrl+L to clear search box
  3. Keep a notepad ready for notes
  4. Have backup searches written down
  5. Practice closing/opening app once


DURING PRESENTATION - TIPS
═══════════════════════════

Good Practices:
✅ Make eye contact with audience, not just screen
✅ Point to elements with mouse (audiences like seeing cursor)
✅ Speak clearly and slowly (you'll naturally speed up)
✅ If something breaks, stay calm and show backup screenshot
✅ If slow, use it as teaching moment: "This is calling Google's AI..."
✅ Pause between sections for questions

If Something Goes Wrong:
❌ Don't panic or apologize excessively
❌ Quickly pivot: "Let me show you a screenshot..."
❌ Continue with explanation: "This would show..."
❌ Always have presenter notes with talking points

Timing Guidelines:
- Problem statement: 1 minute
- Demo: 5-7 minutes
- Technical explanation: 2 minutes
- Closing: 30 seconds
- Q&A: 5-10 minutes
TOTAL: ~15 minutes (adjust based on your schedule)
